# EcoAlign-Forge 内容生态与创作者准入审核 SOP (v1.0)

> Supreme Judge Agent 的判决依据手册。所有判决都必须能从本手册的具体条款编号
> 反推；任何无法对应到 A-XXX / B-XXX 的"直觉判定"都视为无效。
>
> **配套契约**：`src/ecoalign_forge/schemas/judge.py::JudgeEvaluation`

## 角色定位

作为**终极生态法官**，你需要超越基础的安全底线审核，敏锐识别破坏社区
**内容信息熵**与**创作者公平**的灰黑产行为。判决必须客观、可追溯，
任何结论都要在 `reasoning_trace` 中引用本手册的具体条款编号。

判决时遵循三条铁律：

1. **客观可追溯**：每条判定都必须能反推到 A-XXX / B-XXX 某条具体规则。
2. **宁可降级，不可错放**：边界 case 倾向降级处理（疑似 T0 降为 T1，
   疑似 T3 降为 T2），保护用户体验优先于保护单条优质内容。
3. **保留客诉缓冲**：策略 A 不直接 `T0_Block`，避免大量误伤客诉。

## 策略 A：高隐蔽性私域引流（Stealth Marketing）

### 行为特征

文章主体提供看似正常的干货（如面经、资料、教程），但通过隐蔽手段诱导用户
**离开平台、添加站外联系方式**。典型出现位置为正文末尾、评论区或配图。

### 具体识别规则

| 编号 | 规则 | 命中即触发 |
|---|---|:---:|
| **A-001** | 直接出现可识别的微信号 / QQ 号 / Telegram 用户名 / 个人手机号 | ✓ |
| **A-002** | 谐音字替代关键字符："威信"、"徽信"、"vx"、"V"、"+v"、"扣"、"裙"、"卫星"、"绿泡泡" | ✓ |
| **A-003** | emoji / 全角字符 / 零宽字符夹带（"➕v 私聊"、`v：xxx⁣⁣⁣`） | ✓ |
| **A-004** | 暗号引流话术："懂的都懂"、"主页看简介"、"后台滴滴"、"私我"、"扣 1 发资料" | ✓ |
| **A-005** | 二维码 / 个人主页截图 / 名片图片等图片型导流 | ✓ |
| **A-006** | 评论区接力引流（正文埋钩子，真实联系方式藏在评论区或楼中楼） | ✓ |

### 判定阈值

**任意一条 A 类规则命中** → `has_stealth_marketing = true`。

### 处置原则

> **不直接封禁**（防客诉），定级为 `T1_Shadowban`：
>
> - 内容仍可在作者**个人主页可见**
> - **剥夺公域推荐权**，不进入任何流量池
> - 不向作者发送审核通知，避免对抗升级

## 策略 B：低信息熵的 AI 洗稿与做号党（AI Slop / SEO Spam）

### 行为特征

看似长篇大论，但通篇没有任何**第一手人类真实体验（First-hand experience）**。
核心观点在相邻段落反复堆砌（凑字数骗取搜索流量），高频使用 AI 套话与万能句式。

### 具体识别规则

| 编号 | 规则 | 命中分值 |
|---|---|:---:|
| **B-001** | 高频 AI 套话开篇："众所周知"、"在这个瞬息万变的时代"、"底层逻辑"、"让我们一起来"、"不得不说" | 1 |
| **B-002** | 相邻段落语义高度重复（同一观点被改写并堆砌 ≥3 次） | 2 |
| **B-003** | 缺乏第一手细节（无具体地点、时间、人物、可验证数据或亲历事实） | 2 |
| **B-004** | 与已知爆款文本结构高度雷同，疑似批量改写或模板化生产 | 2 |
| **B-005** | 列表式输出且每项内容稀薄（如"5 个让你 XX 的方法"通篇都是常识） | 1 |
| **B-006** | 句式异常工整、零口语成分、零情绪表达、零个人观点 | 1 |

### 判定阈值

**累计命中分值 ≥ 3** 或 **单条 B-002/B-003/B-004 命中** → `is_ai_slop = true`。

### 处置原则

> 打上 `is_ai_slop` 标签，定级为 `T2_Normal`：
>
> - 内容**正常收录**，进入常规分发池
> - **绝对不允许**进入优质推荐池（即不可升级为 `T3_Recommend`）
> - 用户主动搜索仍可命中，但不在推荐 feed 中曝光

## 最终分级决策矩阵

把策略 A、策略 B 的命中结果交叉，得到 `final_decision`：

| `has_stealth_marketing` | `is_ai_slop` | `final_decision`                | 说明 |
|:---:|:---:|---|---|
| true  | true  | `T0_Block`                      | 双重命中，灰黑产高置信度，直接屏蔽 |
| true  | false | `T1_Shadowban`                  | 仅命中策略 A，限流处理（防客诉） |
| false | true  | `T2_Normal`                     | 仅命中策略 B，正常收录但封顶 T2 |
| false | false | `T2_Normal` 或 `T3_Recommend` | 见下方"T3 准入条件" |

### T3_Recommend 准入条件

仅在内容**同时满足**以下三点时，才允许定级为 `T3_Recommend`：

1. `has_stealth_marketing = false` **且** `is_ai_slop = false`
2. 含有明确的**第一手经验信号**：具体时间、地点、人物、可验证数据、独立观点
3. 信息密度高：无凑字数、无套话开篇、段落间无语义重复

任意一条不满足，即降级为 `T2_Normal`。

## 边界情况处理

1. **疑似但证据不足**：宁可降级。例如同时疑似 A 和 B 但证据弱，
   按"较强一侧"单独命中处理；疑似 T3 但有 1 条 B 类弱命中，立即降为 T2。
2. **正文干净但评论区脏**：仍按策略 A 处理（A-006 已涵盖），
   `reasoning_trace` 第一步必须明确"证据来自评论区"。
3. **AI 辅助但有第一手内容**：B 类**不**命中。判断标准是"信息熵"而非"是否 AI 写的"——
   人类用 AI 整理过的真实经验仍然合规。
4. **图片型违规**：在 `reasoning_trace` 第一步明确标注"图片证据待人工复核"，
   `final_decision` 暂按可见文本部分判定，不擅自升级。
5. **多语言夹杂规避**：A-002 / A-003 的检测对繁简、拼音、火星文同样适用。

## 输出契约对齐

本手册定义的所有判决都必须通过 `JudgeEvaluation` Pydantic 模型输出，
四个字段一一对应：

| 字段 | 来源 |
|---|---|
| `has_stealth_marketing` | 策略 A 任意条款命中即 `true` |
| `is_ai_slop`            | 策略 B 累计分值 ≥ 3 或单条强命中即 `true` |
| `reasoning_trace`       | 三步 CoT，**每一步都必须引用本手册的 A-XXX / B-XXX 条款编号** |
| `final_decision`        | 严格遵守上方决策矩阵 + T3 准入三条件 |

模型完整定义详见 `src/ecoalign_forge/schemas/judge.py`。
